# Hybrid Video Analysis System V2 — 开发文档

> 最后更新: 2026-04-12

---

## 一、项目概述

本系统是一个基于 YOLO + Qwen-VL 的混合视频分析平台，支持同时接入多路 RTSP 视频流，并行执行多种 AI 检测任务，将事件结果持久化到 MongoDB + MinIO，并通过 REST API 和 Vue.js 前端进行管理。

### 核心能力

| 功能 | 需求编号 | 状态 |
|------|---------|------|
| 违停电子围栏检测 | FR1 | 已实现 |
| 烟火检测（YOLO + Qwen-VL 二次验证） | FR2 | 已实现 |
| 公共空间利用分析（Qwen-VL） | FR3 | 已实现 |
| 多路 RTSP 并行处理 + 任务组合 | FR4 | 已实现 |
| 事件持久化与查询 | FR5 | 已实现 |
| 前端 WebUI（仪表盘/流管理/事件/上传） | FR6.1/6.2 | 已实现 |
| Webhook 事件回调通知 | FR6.3 | 已实现（本次新增） |
| 停留时间违停判定（连续N帧） | NFR2.1 | 已实现（本次新增） |

## 四、系统架构

### 4.1 数据流

```
RTSP 视频流
    │
    ▼
VideoFrameCapture (帧捕获, 每流独立线程)
    │  batch_callback (每批8帧或1秒)
    ▼
StreamManager._make_callback()
    │  frame.source_id = camera_id
    ▼
StreamRuntime.dispatch_frame()
    │
    ▼
StreamExecutor.dispatch()
    │  ThreadPoolExecutor (每流独立线程池)
    │  并行分发到各任务 handler
    ├──────────────────────────┬──────────────────────┐
    ▼                          ▼                      ▼
ViolationDetection         SmokeFlameDetection    CommonSpaceDetection
 │                          │                      │
 ├─ YOLO 推理               ├─ YOLO 推理            ├─ Qwen-VL API 调用
 ├─ 区域过滤                 ├─ Qwen-VL 二次验证      │
 ├─ DwellTracker 停留判定     │                      │
 ├─ 渲染标注帧               ├─ 渲染标注帧            │
 ├─ MinIO 上传              ├─ MinIO 上传           ├─ MinIO 上传
 └─ MongoDB 保存            └─ MongoDB 保存         └─ MongoDB 保存
         │                          │                      │
         └──────────────────────────┴──────────────────────┘
                                    │
                              event_generator.py
                              handle_event_detected()
                                    │
                                    ▼
                           WebhookService.notify()
                           (异步POST到注册的URL)
```

### 4.2 隔离模型

每条 RTSP 流拥有独立的：
- `StreamRuntime` 实例
- `StreamExecutor` 线程池 (max_workers = max(2, task_count * 2))
- `YOLOModelLoader` + 模型实例
- 各任务 handler 实例（ViolationDetectionService 等）
- `DwellTracker` 实例（每个 source_id 一个）

共享的（进程级 `AppResources`）：
- MongoDB 连接
- MinIO 连接
- 禁停区配置 (NoParkingZoneChecker)
- Qwen-VL 客户端

## 六、核心模块详解

### 6.1 DwellTracker — 停留时间违停判定

**文件:** `backend/services/dwell_tracker.py`

**设计思路:** 需求文档要求"违停应该是车辆停下来，而不是禁止通行"，因此改为连续 N 帧中心点在禁停区内才触发事件。

**关键参数:**
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `dwell_threshold` | 5 | 连续多少帧触发违停（env: `DWELL_THRESHOLD`） |
| `_MATCH_DISTANCE_THRESHOLD` | 80px | 帧间中心点匹配最大距离 |
| `_MISSING_EXPIRE_FRAMES` | 3 | 多少帧未出现则过期 |

**工作流程:**
1. 接收当前帧已在禁停区内的检测结果
2. 计算每个检测框中心点 (cx, cy)
3. 与现有 tracks 做贪心最近邻匹配（按距离排序）
4. 匹配上的 track: `dwell_frames += 1`
5. 未匹配的检测: 创建新 track
6. 未匹配的 track: `missing_frames += 1`，超过 3 帧过期
7. `dwell_frames >= threshold` 且未报告过 → 触发违停事件
8. 同一辆车持续停留只报一次（`reported = True`），离开后重置

**在 violation_detection.py 中的集成:**
```python
# 原来: zone过滤 → 有违规即上传保存
# 现在: zone过滤 → DwellTracker.update() → 仅停留达标的才上传保存
in_zone = self.zone_checker.filter_violations_in_zones(detections, source_id)
violations = tracker.update(in_zone)  # 只返回达到停留阈值的
if not violations:
    return  # 未达标，跳过本帧
```

### 6.2 WebhookService — 事件回调通知

**文件:** `backend/services/webhook_service.py`

**设计思路:** 检测事件保存到 MongoDB 后，异步 POST 通知到外部系统，不阻塞检测流水线。

**关键参数:**
| 参数 | 值 | 说明 |
|------|-----|------|
| 线程池 | 4 workers | 异步发送，不阻塞主线程 |
| 缓存 TTL | 60 秒 | 避免每次事件都查 MongoDB |
| 请求超时 | 5 秒 | 单次 POST 超时 |
| 最大重试 | 2 次 | 失败后最多重试 2 次 |

**数据流:**
```
event_generator.handle_event_detected()
  → mongo_client.save_event(event)  # 成功后
  → _webhook_service.notify({...})  # fire-and-forget
      → ThreadPoolExecutor
          → _post_with_retry(url, payload)
```

**MongoDB 集合:** `webhook_configs`
```json
{
  "_id": ObjectId("..."),
  "url": "https://external-system.com/callback",
  "event_types": ["parking_violation"],
  "created_at": 1672531200.0
}
```

### 6.3 StreamManager — 流生命周期管理

**文件:** `backend/services/stream_manager.py`

- 负责 RTSP 流的增删改查
- 通过 `StreamRuntimeFactory` 为每条流创建隔离运行时
- 流配置持久化到 MongoDB `streams` 集合，重启自动恢复
- 添加带 `parking_violation` 任务的流时，自动检查禁停区配置，若缺失则弹出 GUI 绘制工具

### 6.4 ViolationDetectionService — 违停检测流水线

**文件:** `backend/services/violation_detection.py`

完整流水线:
1. **YOLO 推理** — `yolov8n.pt` 检测车辆（串行锁保护）
2. **区域过滤** — `NoParkingZoneChecker.filter_violations_in_zones()`
3. **停留判定** — `DwellTracker.update()` 连续 N 帧停留才触发
4. **渲染标注** — `render_official_frame()` 画框 + 禁停区
5. **MinIO 上传** — 渲染后的帧上传为 JPEG
6. **MongoDB 保存** — 通过 `event_generator` 写入事件文档

### 6.5 EventModel — 事件数据模型

**文件:** `backend/models/event.py`

Pydantic 模型，存入 MongoDB `events` 集合:

| 字段 | 类型 | 说明 |
|------|------|------|
| `camera_id` | str | 摄像头 ID |
| `timestamp` | float | Unix 时间戳 |
| `event_type` | str | 事件类型 |
| `image_url` | str | MinIO 图片 URL |
| `confidence` | float | 置信度 [0, 1] |
| `bbox` | (x1,y1,x2,y2) | 边界框（可选） |
| `object_count` | int | 检测目标数（可选） |
| `objects` | list | 目标明细列表（可选） |
| `description` | str | 人类可读描述 |
| `detection_stage` | str | 检测阶段 |
| `zone_polygon` | list | 禁停区坐标 |
| `analysis_result` | str | AI 分析文本（公共空间） |
| `analysis_summary` | dict | 结构化摘要（公共空间） |
| `metadata` | dict | 扩展元数据 |
